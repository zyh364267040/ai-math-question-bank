# AI 数学题库项目第一版方案

## 一、项目说明

本项目首先是一套服务于日常数学教学的个人实用题库，同时也是学习和实践 AI 产品经理能力的真实项目。

当前上课时缺少可以持续使用的题目资源，经常需要临时、重复寻找题目。本项目计划把平时获得的 PDF、Word、图片等试题逐步电子化、分类并长期保存，形成可以持续积累的个人数学题库，方便以后备课、上课和复习时快速选题，避免重复寻找同类题目。

题库中的题目也可以用于给学生进行分类练习，帮助学生熟悉和掌握不同知识点与题型。

现阶段以讨论和需求梳理为主。Hermes 负责持续整理方案；需要后续决定的内容统一标记为“待讨论”。

## 二、项目定位

### 1. 主要用户

- 第一使用者：本人，用于备课、上课和选题
- 第二使用者：学生，用于不同知识点和题型的练习
- 地区范围：第一阶段优先服务天津地区

### 2. 主要解决的问题

- 上课前需要临时寻找合适的题目
- 同类题目可能被反复寻找，浪费备课时间
- 已经使用过的好题没有形成统一积累
- PDF 中的题目不方便单独提取和重复使用
- 难以按知识点、题型和难度快速找题
- 题目来源、年份和考试信息容易丢失
- 难以记录哪些题目已经讲过或使用过
- 后续组卷、整理错题和给讲义配题时缺少统一题库

### 3. 第一阶段核心目标

把以后下载或获得的题目转换成可编辑的电子版，分类保存到个人题库中，并支持快速查找和导出。

第一阶段先验证“试卷导入、题目电子化、分类入库、查询选题、导出 Markdown”这一完整流程。

## 三、题目来源

计划收集：

- 天津高考真题
- 天津各区一模、二模试卷
- 学校月考、期中和期末试卷
- 教材及公开资料
- 各类教辅材料
- 教师自编题
- 学生错题
- 其他后续获得的试题资料

每道题需要保留以下来源信息：

- 地区
- 年份
- 考试类型
- 试卷名称
- 原始题号
- 原始页码
- 原始文件位置

## 四、题目电子化流程

初步流程：

> 获取 PDF、Word 或图片 → 保存原始文件 → 按题切分 → OCR 识别 → 数学公式转为 LaTeX → 处理题目配图 → AI 自动分类和打标签 → 人工审核 → 正式入库

当前确定：

- 试卷主要以 PDF 文件为主
- 题目需要转换为可编辑的 Markdown 内容
- 数学公式使用 LaTeX 保存
- 题目配图需要清晰、完整并精准切割
- 原图质量差或不适合使用时，允许重新绘图
- Codex 可以承担 PDF 处理、OCR、切题、配图处理和批量入库等执行工作，但正式采用前需要用样本验证效果

原始题目保存原则：

- 完整的原始 PDF 需要保留，作为追溯和核对依据
- 正式入库后，不强制长期保存每道题的整题截图，避免仓库过度膨胀
- 几何图、函数图像等题目必需配图需要单独保留
- OCR 审核阶段可以临时保留切题截图，审核完成后再按需要清理

## 五、保存方式

采用“原始资料、题目资源、结构化数据库”三层保存。

### 1. 原始资料

保存完整的 PDF、Word 和图片原件，不直接修改，方便追溯和核对。

### 2. 题目资源

保存题目必需的几何图、函数图像和其他插图。审核期间可以临时保存整题截图，审核完成后根据实际需要清理。

### 3. 结构化数据库

题目文字、答案、解析、来源和标签保存在 SQLite 数据库中，通过题目编号与题目配图和原始试卷关联。

每道题分配唯一编号，例如：`TJ-2025-GK-MATH-001`。

## 六、每道题需要保存的信息

### 1. 题目内容

- 题目编号
- 题干
- 数学公式
- 选项
- 各小问
- 题目配图
- 标准答案
- 详细解析

### 2. 来源信息

- 地区
- 年份
- 考试类型
- 试卷名称
- 原始题号
- 分值
- 原始文件位置
- 原始页码

### 3. 分类标签

- 学段和年级
- 一级模块
- 二级知识点
- 三级知识点
- 题型
- 难度
- 解题方法
- 易错点
- 关联知识点
- 适用场景

### 4. 质量状态

- OCR 是否完成
- 公式是否检查
- 答案是否检查
- 解析是否检查
- 标签是否人工确认
- 是否疑似重复题
- 是否允许正式使用

### 5. 使用记录

- 是否已经讲过或使用过
- 被选用次数
- 最近使用时间
- 学生作答次数
- 正确率
- 平均用时
- 常见错误

第一版可以优先记录“是否使用、使用次数和最近使用时间”，学生作答数据可在后续阶段逐步增加。

## 七、不同题型的保存规则

- 选择题：题干、选项、答案和解析分别保存
- 填空题：题干、标准答案和解析分别保存
- 解答题：保存完整母题，每个小问单独记录并关联母题
- 含图题：题目文字与配图文件关联
- 综合题：允许同时关联多个知识点

题型和知识点主要通过数据库标签管理，不依靠大量文件夹分类。

## 八、考试频率统计

考试频率不由人工凭感觉填写，而是根据已收录题目的来源和标签自动统计。

第一阶段重点统计天津高考频率，包括：

- 某知识点近 5 年、近 10 年出现次数
- 天津高考出现次数
- 常见题型和试卷位置
- 常见分值和难度
- 最近一次出现时间

一模、二模和校内考试频率可以保留数据，后续再决定是否重点分析。全国卷暂不作为第一阶段重点。

## 九、AI 能力设想

第一阶段可能使用：

- PDF 自动拆题
- 题目 OCR 与公式识别
- 题目配图自动切割
- 自动识别知识点和题型
- 自动判断初步难度
- 自动识别解题方法和易错点
- 重复题识别
- 自然语言搜题
- 按要求选择题目并生成 Markdown 练习

AI 输出不能直接视为正确结果。公式、答案、解析、标签和重新绘制的图形都需要设置审核环节。

## 十、项目协作分工

本项目采用 Hermes 与 Codex 协作开发：

- Hermes：理解需求、产品规划、任务拆分、制定验收标准、检查 Codex 产物、整理讨论结论
- Codex：创建项目、编写代码、处理 PDF/OCR/图片、批量文件操作、运行测试和修复问题
- ChatGPT：需要时参与产品复盘、体验评价和改进建议

执行原则：Hermes 不默认代替 Codex 完成项目开发。每个开发阶段先由 Hermes 明确任务、输入、输出和验收条件，再调用真实 Codex CLI 执行；Hermes 最后读取文件、运行测试并核验结果。如果 Codex 未运行、被权限阻塞或执行失败，必须如实说明，不把 Hermes 或其他子代理的执行描述成 Codex 完成。

## 十一、第一版产品范围

建议先使用少量真实试卷验证完整流程，不立即建设大规模题库。

暂定范围：

- 地区：天津
- 学科：高中数学
- 资料数量：3～5 份真实试卷
- 题目数量：30～50 道
- 第一版导出格式：Markdown
- 第一批专题：开始实施时再确定
- 第一批试卷：由本人寻找和提供

第一版需要实现：

1. 导入 PDF
2. 自动拆分题目
3. OCR 与公式识别
4. 处理题目配图
5. AI 初步分类和打标签
6. 人工审核
7. 保存到个人题库
8. 按条件查找题目
9. 导出 Markdown 练习

## 十二、需要重点考虑的问题

- 数学公式 OCR 的准确性
- 题目切分是否完整
- 题目配图是否清晰准确
- 重新绘图是否与原题一致
- 答案和解析是否正确
- AI 标签是否可靠
- 同一道题的重复识别
- 原始资料和教辅题目的版权
- 本地模型与云端模型的成本
- 数据备份和迁移
- 后续扩充时标签是否统一
- 如何防止数据库和图片资源越来越臃肿

## 十三、第一版成功标准

1. 能从 PDF 稳定拆分并电子化题目
2. 数学公式和配图能够满足实际使用要求
3. 能准确保存题目来源和分类标签
4. 能按知识点、题型和难度快速找到题目
5. 能记录题目是否已经使用，减少重复找题
6. 能生成一份可直接使用的 Markdown 专题练习
7. 整体流程比人工找题和整理更省时间

## 十四、当前已确定事项

- 项目首先服务于个人教学，其次作为 AI 产品经理实践项目
- 第一阶段主要由本人使用，同时为学生提供分类练习
- 优先建设天津高中数学题库
- 试题资料以 PDF 为主
- 原始试卷需要保留
- 题目正文以 Markdown 保存，公式使用 LaTeX
- 题目文字和标签进入 SQLite 数据库
- 题目必需配图单独保存并与题目关联
- 不强制长期保存每道题的整题截图
- 每道题需要分配唯一编号
- 每道题需要记录知识点、题型、来源和使用情况
- 知识点采用稳定 ID，AI 只能从现有体系选择，不得自造标签
- 考试频率由系统根据题目数据自动统计
- 考频为统计结果，不作为单题人工标签
- 第一版先导出 Markdown
- 第一批专题和试卷在正式开始时确定

## 十五、待讨论事项

- 难度如何划分

- Codex 处理 PDF、图片和重新绘图的实际效果
- 如何判断两道题是否属于重复题或变式题
- 第一版查询和选题界面采用什么形式
- 第一版是否加入学生错题功能

- 数据库如何备份和迁移
