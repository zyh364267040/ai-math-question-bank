# 题目数据字段与标签标准

## 一、文档用途

本标准规定每道题入库时保存的数据、必填范围、产生方式和审核要求，作为下一步 SQLite 表结构、约束、字典表和关联表设计的直接依据。

字段的“必填”指题目进入正式可用状态时必须具备；草稿和待审核状态允许暂缺。多值字段在数据库中应使用关联表，不保存为逗号拼接文本。字典值应保存稳定代码，中文名称用于展示。

产生方式统一使用“系统、AI、人工、统计”：系统指程序自动生成或维护；AI 指模型生成的候选结果；人工指录入或审核确认；统计指从已有记录聚合计算。标为“AI + 人工”的字段表示 AI 可预填，但必须由人工确认后才能正式使用。

## 二、设计原则

- 教学实用优先：字段服务于备课、选题、讲题、练习和复用，避免为了理论完整增加首版录入负担。
- 天津高考优先：地区、考试类型和统计口径首先满足天津高考及天津模拟考试，字段结构仍保留扩展能力。
- 原始来源可追溯：每道来源题必须能定位到原始文件、试卷、题号和页码；教师自编题等无文件来源的题目须明确来源类型。
- AI 结果需审核：AI 生成或识别的正文、公式、答案、解析和标签均为候选值，未经人工审核不得进入正式可用状态。
- 考频由数据计算：考频不是单题固定标签，不人工填写；由地区、年份、考试类型、知识点和题型等维度组合统计。
- 稳定标识优先：业务编号、知识点 ID、标签代码一经正式使用不因展示名称调整而改变。
- 原始事实与推断分离：来源、原题内容等事实字段与 AI 分类、难度判断等推断字段分别保存和审核。

## 三、题目唯一编号规则

### 1. 业务编号

格式为 `地区-年份-考试类型-学科-流水号`，示例为 `TJ-2025-GK-MATH-001`。

- 地区使用受控代码，如 `TJ` 表示天津；全国或来源地区不明确时使用经字典确认的代码，不临时自造。
- 年份使用四位公历年份；无法确认时使用 `0000`，并保持年份字段为空，待人工补全。
- 考试类型使用受控代码，如高考 `GK`、一模 `YM`、二模 `EM`、月考 `YK`、期中 `QZ`、期末 `QM`、教材 `JC`、教辅 `JF`、教师自编 `ZB`、学生错题 `CT`、其他 `QT`。
- 学科首版固定为 `MATH`。
- 流水号在“地区 + 年份 + 考试类型 + 学科”范围内从 `001` 递增；超过三位时自然扩展，不复用已作废编号。
- 同一母题的各小问不另行改变业务编号；小问通过独立内部 ID、顺序号和母题关系保存。若小问需要独立检索，可展示为业务编号加小问后缀，如 `TJ-2025-GK-MATH-001-2`，该后缀不是新的母题业务编号。
- 业务编号在首次正式入库时由系统生成，创建唯一约束；来源信息后来纠正时原则上不改号，确需改号须保留编号变更记录。

### 2. 数据库内部 ID 与业务编号的区别

- 数据库内部 ID `id` 是表关联使用的无业务含义主键，建议使用 SQLite `INTEGER` 自增主键。它稳定、高效，但不展示给用户，也不承诺跨数据库迁移后仍相同。
- 业务编号 `question_code` 是人可读、可检索、可导出和跨系统引用的唯一编号，使用 `TEXT` 并设置唯一约束。
- 所有外键关系优先引用内部 ID；导入导出、文件命名和人工沟通优先使用业务编号。

## 四、字段定义

### 1. 核心标识

- 英文名：`id`；中文名：数据库内部 ID；建议数据类型：`INTEGER`；是否必填：是；产生方式：系统；说明：题目表主键，仅用于数据库关联。
- 英文名：`question_code`；中文名：题目业务编号；建议数据类型：`TEXT`；是否必填：是；产生方式：系统；说明：按唯一编号规则生成，设置唯一约束。

### 2. 题目正文

- 英文名：`stem_markdown`；中文名：题干；建议数据类型：`TEXT`；是否必填：是；产生方式：AI + 人工；说明：保存可编辑 Markdown，行内与独立公式均在正文中用 LaTeX 引用。
- 英文名：`options`；中文名：选项；建议数据类型：关联子表；是否必填：条件必填；产生方式：AI + 人工；说明：单选、多选题必填；每项保存选项代码、内容和顺序，不拼接为一个文本字段。
- 英文名：`subquestions`；中文名：小问；建议数据类型：题目自关联或小问子表；是否必填：否；产生方式：AI + 人工；说明：每小问保存顺序号、题干、答案、解析和分值；必须关联母题。
- 英文名：`formula_latex`；中文名：公式；建议数据类型：关联子表；是否必填：否；产生方式：AI + 人工；说明：用于公式级检索、定位和审核，保存 LaTeX、出现位置及顺序；题干中的公式仍保留原位。
- 英文名：`figures`；中文名：配图；建议数据类型：关联资源表；是否必填：条件必填；产生方式：AI + 人工；说明：原题依赖配图时必填，保存相对路径、用途、顺序、替代文本、来源类型和图像哈希。
- 英文名：`answer_markdown`；中文名：答案；建议数据类型：`TEXT`；是否必填：是；产生方式：AI + 人工；说明：保存标准答案；多小问题目还应在小问级保存对应答案。
- 英文名：`analysis_markdown`；中文名：解析；建议数据类型：`TEXT`；是否必填：否；产生方式：AI + 人工；说明：保存可编辑的详细解析；首版允许暂缺，但若存在必须审核后使用。

### 3. 来源信息

- 英文名：`region_code`；中文名：地区；建议数据类型：`TEXT`；是否必填：是；产生方式：AI + 人工；说明：引用地区字典，首版重点为天津 `TJ`。
- 英文名：`exam_year`；中文名：年份；建议数据类型：`INTEGER`；是否必填：条件必填；产生方式：AI + 人工；说明：试卷来源题必填；教师自编且无明确年份时可空。
- 英文名：`exam_type_code`；中文名：考试类型；建议数据类型：`TEXT`；是否必填：是；产生方式：AI + 人工；说明：引用考试类型字典，不保存自由文本。
- 英文名：`paper_name`；中文名：试卷名；建议数据类型：`TEXT`；是否必填：条件必填；产生方式：AI + 人工；说明：来自试卷时必填，使用原始资料可核对的规范全名。
- 英文名：`source_question_no`；中文名：题号；建议数据类型：`TEXT`；是否必填：条件必填；产生方式：AI + 人工；说明：来自试卷时必填，保留原卷表示，如 `18` 或 `18（2）`。
- 英文名：`source_page`；中文名：页码；建议数据类型：`TEXT`；是否必填：否；产生方式：AI + 人工；说明：可保存单页或页码范围；按原始文件页序，不按 PDF 阅读器外显页码推测。
- 英文名：`score`；中文名：分值；建议数据类型：`REAL`；是否必填：否；产生方式：AI + 人工；说明：单位为分；多小问分值可同时保存在小问级。
- 英文名：`source_file_path`；中文名：原始文件相对路径；建议数据类型：`TEXT`；是否必填：条件必填；产生方式：系统 + 人工；说明：有原始文件时必填，必须保存相对项目资料根目录的路径，禁止保存仅本机有效的绝对路径。

### 4. 知识点

- 英文名：`primary_knowledge_point_id`；中文名：主知识点；建议数据类型：`INTEGER` 外键；是否必填：是；产生方式：AI + 人工；说明：只能引用《高中数学高考知识点体系》维护的稳定知识点 ID，且必须定位到三级知识点；AI 只能从候选列表选择，不得生成新 ID 或新知识点。主知识点只能一个。
- 英文名：`related_knowledge_point_ids`；中文名：关联知识点；建议数据类型：多对多关联表；是否必填：否；产生方式：AI + 人工；说明：可多选，同样只能引用现有稳定 ID；不得与主知识点重复。

知识点表应独立维护 `id`、体系编号、名称、层级、父级 ID、启用状态和体系版本。现有体系中的引用路径可展示为“08.06 导数与恒成立—分离参数”，但题目关联实际保存知识点表稳定 ID，不以可变名称作为外键。新增知识点必须先经人工更新知识点体系和字典，再供 AI 选择。

### 5. 分类标签

- 英文名：`question_type_code`；中文名：题型；建议数据类型：`TEXT`；是否必填：是；产生方式：AI + 人工；说明：引用题型字典，只能单选。
- 英文名：`difficulty_level`；中文名：难度；建议数据类型：`INTEGER`；是否必填：否；产生方式：AI + 人工；说明：取 1 至 5；首版为初步判断，必须保留后续校准能力。
- 英文名：`task_codes`；中文名：解题任务；建议数据类型：多对多关联表；是否必填：否；产生方式：AI + 人工；说明：描述题目要求完成什么；允许多选，但应只选题目明确或核心的任务。
- 英文名：`method_codes`；中文名：解题方法；建议数据类型：多对多关联表；是否必填：否；产生方式：AI + 人工；说明：可多选，只记录解析中实际采用或可稳定复用的方法，不记录泛化空标签。
- 英文名：`error_tags`；中文名：易错点；建议数据类型：多对多关联表；是否必填：否；产生方式：AI + 人工；说明：可多选；首版可保存受控代码加简短补充说明，后续根据真实题沉淀字典。
- 英文名：`usage_scenario_codes`；中文名：适用场景；建议数据类型：多对多关联表；是否必填：否；产生方式：AI + 人工；说明：可多选，表示教学用途，不代表实际使用记录。

### 6. 质量审核

审核项统一建议使用状态值 `pending`（待审核）、`passed`（通过）、`rejected`（不通过）、`not_applicable`（不适用）。不得用单一布尔值掩盖“尚未审核”和“不适用”的区别。

- 英文名：`ocr_review_status`；中文名：OCR 审核状态；建议数据类型：`TEXT`；是否必填：是；产生方式：人工；说明：核对文字是否完整准确；纯人工录入可标记不适用。
- 英文名：`formula_review_status`；中文名：公式审核状态；建议数据类型：`TEXT`；是否必填：是；产生方式：人工；说明：核对 LaTeX、符号、上下标、区间和公式位置；无公式可标记不适用。
- 英文名：`figure_review_status`；中文名：配图审核状态；建议数据类型：`TEXT`；是否必填：是；产生方式：人工；说明：核对清晰度、完整性、裁切和重绘一致性；无图可标记不适用。
- 英文名：`answer_review_status`；中文名：答案审核状态；建议数据类型：`TEXT`；是否必填：是；产生方式：人工；说明：答案通过后题目才可正式使用。
- 英文名：`analysis_review_status`；中文名：解析审核状态；建议数据类型：`TEXT`；是否必填：是；产生方式：人工；说明：无解析时标记不适用；有解析时核对结论、推导和方法。
- 英文名：`tag_review_status`；中文名：标签审核状态；建议数据类型：`TEXT`；是否必填：是；产生方式：人工；说明：至少核对主知识点和题型，正式可用前必须通过。
- 英文名：`usability_status`；中文名：整体可用状态；建议数据类型：`TEXT`；是否必填：是；产生方式：系统 + 人工；说明：建议取 `draft`、`pending_review`、`usable`、`needs_fix`、`disabled`；只有必要审核项通过后才能置为可用。
- 英文名：`reviewer`；中文名：审核人；建议数据类型：`TEXT`；是否必填：条件必填；产生方式：系统 + 人工；说明：发生人工审核时必填；后续有用户表时改为审核人 ID。
- 英文名：`reviewed_at`；中文名：审核时间；建议数据类型：`TEXT`；是否必填：条件必填；产生方式：系统；说明：审核状态提交时记录 ISO 8601 时间。
- 英文名：`review_notes`；中文名：审核备注；建议数据类型：`TEXT`；是否必填：否；产生方式：人工；说明：记录错误、修订依据和遗留风险；不通过时建议必填。

使用汇总字段可以查询时计算或缓存，但使用记录子表是事实来源。适用场景是对未来用途的标签，使用记录是已经发生的事件，两者不得混用。

## 五、标签字典第一版

### 1. 题型

题型只能单选：

- `single_choice`：单选
- `multiple_choice`：多选
- `fill_blank`：填空
- `solution`：解答

### 2. 难度

难度使用 5 级初稿，以目标学生在限定时间内完成题目所需的知识跨度、推理步骤和运算负担综合判断：

- 1 级“基础识记”：直接使用单一概念、公式或基本运算，步骤短，干扰少。
- 2 级“基础应用”：单一主知识点的常规变式，通常需要两至三个明确步骤。
- 3 级“中等综合”：需要知识点联结、一次关键转化或较完整推理，运算量中等。
- 4 级“较难综合”：跨知识点、多阶段推理或明显分类讨论，对策略选择要求较高。
- 5 级“压轴探究”：结构隐蔽、参数复杂、存在性或开放探究突出，需要多次关键转化。

当前方案只是可执行初稿，不代表天津高考难度的最终标尺。应以首批真实题的教师判断、题位、分值、作答表现和耗时进行校准；校准前不把 AI 难度作为无须复核的事实。

### 3. 解题任务

第一版字典包括：求值、求解、证明、判断、参数范围、最值、取值范围、零点个数、存在性、轨迹、定点、定值、计数、概率计算、模型建立。可根据真实题经人工审核扩充，不允许 AI 临时自造代码。

### 4. 解题方法

第一版字典包括：数形结合、分类讨论、等价转化、构造函数、换元、韦达定理、空间向量、概率建模、函数与方程、分离参数、配方法、基本不等式、判别式、特殊值法、反证法、裂项相消、错位相减。方法可多选，但必须能在解析或审核备注中找到依据。

### 5. 易错点

易错点首版采用“受控标签 + 可选补充说明”。初始标签包括：概念混淆、条件遗漏、分类不全、定义域遗漏、取值范围遗漏、符号错误、公式误用、运算错误、图形关系误判、答案格式错误。后续只根据真实题中的稳定共性扩充。

### 6. 适用场景

第一版字典包括：课堂例题、课堂练习、课后作业、专题复习、测试组卷、错题巩固。

### 7. 多标签规则

- 主知识点只能一个，且必须选择。
- 关联知识点可以多选。
- 题型只能单选。
- 解题任务可以多选，但应突出题目实际要求。
- 解题方法可以多选。
- 易错点可以多选。
- 适用场景可以多选。
- 同一标签在同一道题上不得重复关联；关联表应设置“题目 ID + 标签 ID”联合唯一约束。
- AI 只能从当前启用的知识点和标签字典中选择。无法匹配时标记待人工处理，不得自造标签。

## 六、考频统计规则

考频不是单题固定标签，也不是人工审核字段。它由已达到统计纳入条件的题目实时聚合或定期生成统计结果。

第一版统计维度至少包括地区、年份、考试类型、知识点和题型，可按所选维度组合筛选；知识点统计应同时明确“仅主知识点”或“主知识点加关联知识点”的口径。重复题是否去重、母题与小问如何计数也必须在统计任务中指定，不能写回单题标签。

建议统计结果保存统计口径、起止年份、样本题数、出现次数、生成时间和数据版本，以便解释“近 5 年天津高考某知识点出现次数”。原始单题只保存可复算的来源和标签事实。

## 八、第一版最小必填字段

为避免首批 30 至 50 道题录入负担过重，草稿可先只保存题干与来源线索；题目进入“正式可用”时，第一版最小必填集合如下：

- 系统标识：数据库内部 ID、题目业务编号。
- 题目正文：题干；选择题必须有选项；原题依赖配图时必须有关联配图；标准答案。
- 来源信息：地区、考试类型；来自试卷时另须年份、试卷名、题号和原始文件相对路径。
- 核心分类：一个主知识点、一个题型。
- 质量审核：OCR、公式、配图、答案、解析、标签各项状态，整体可用状态；发生审核时须有审核人和审核时间。无公式、无图或无解析时对应项可标记不适用。
- 使用汇总：是否使用、使用次数，默认分别为否和 0。
- 系统维护：创建时间、更新时间、内容哈希、版本状态、版本号。

关联知识点、难度、解题任务、解题方法、易错点、适用场景、详细解析、分值、页码和使用记录均可在首次正式入库后逐步补充。若已有解析，解析必须审核；缺少解析不阻止首版题目用于选题，但导出时应明确无解析。

## 九、对 SQLite 设计的约束

- 题目主表保存单值核心字段；选项、小问、公式、配图、审核历史、使用事件和版本历史使用子表。
- 知识点、解题任务、解题方法、易错点和适用场景使用字典表及关联表；主知识点可在题目表保存外键。
- 地区、考试类型、题型、审核状态和版本状态使用受控代码并设置检查约束或外键。
- 对 `question_code` 设置唯一约束；对 `content_hash`、来源组合、主知识点、题型、可用状态和最近使用时间设置常用查询索引。
- 必填和条件必填应同时由数据库约束与入库校验实现；跨字段条件不适合 SQLite 简单约束时，由入库服务校验并保留审核状态。
- 删除知识点或标签应采用停用状态，不直接物理删除已被题目引用的字典记录。

## 十、仍需用户确认

- 现有三级知识点的稳定 ID 具体采用纯数字主键，还是另加人可读代码；建议数据库用数字主键、展示和导入用稳定代码并存。
- 首版“正式可用”是否强制具备详细解析；本标准暂按答案必填、解析可后补处理。
- 审核人首版使用自由文本，还是立即建立单用户表；个人题库首版建议先用自由文本。
- 重复题统计时按重复组只计一次，还是每张真实试卷均计一次；建议考频按试卷出现事实计数，选题去重时按重复组处理。
- 难度校准优先采用教师判断与题位，还是同步采集学生正确率和用时；首批无学情数据时建议先采用教师判断与题位。
