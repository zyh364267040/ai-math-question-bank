# 整套试卷自动拆题流程

## 目标

系统根据已经确认的题目区域，一次性生成整套“完整单题原图”。用户不参与裁切，只在审核页查看结果。模块不修改原始渲染页、候选题、AI 审核、配图清单或数据库。

## 输入模型

`generate_question_crops` 接收 job 目录、题目区域计划和预期题号。每题由一个或多个 region 组成，每个 region 只含：

- `page_number`：来源页码，必须出现在 `render_manifest.json` 白名单中。
- `bbox`：像素坐标 `[left, top, right, bottom]`，右、下边界不包含在裁切结果中。

单区域直接保存。多区域按输入中的阅读顺序纵向拼接；不同宽度时左对齐，未覆盖区域及分隔带使用白色。清单的 `composition` 会记录单图或纵向拼接方式、区域数和分隔高度。模型不限制页数、题数，也支持跨页区域；实际试卷若没有跨页题，不应人为拆分或伪造跨页。

典型调用：

```python
from src.processing.question_crop import generate_question_crops

manifest = generate_question_crops(
    job_dir=job_dir,
    questions=[
        {
            "question_no": 1,
            "regions": [{"page_number": 1, "bbox": [300, 640, 2050, 850]}],
            "warnings": [],
        }
    ],
    expected_question_nos=[1],
)
```

## 严格校验

生成前会完整读取 `render_manifest.json`，检查任务号、页数、页码唯一性、安全相对路径、PNG 实际格式、像素尺寸和源页 SHA-256。源页实际哈希与清单不一致时立即拒绝处理。

题目计划会检查：题号为正整数且唯一、与预期题号全集完全一致、region 非空、页码属于渲染白名单、bbox 为四个整数且顺序正确并在页面边界内、输出固定为对应的 `question_crops/QNNN.png`、最小宽高以及 warnings 类型。固定文件名既避免题号错配，也阻断绝对路径、反斜线和 `..` 路径穿越。

每张临时图片写入后会用 Pillow 重新打开并验证 PNG 格式和尺寸，再计算字节数与 SHA-256。整套清单写入临时文件后再次 JSON 解析。任何一题失败都会清理临时目录，不会发布半套结果。

## 事务发布与幂等

所有题图先进入 job 目录下的随机临时目录。全部图片和清单验证通过后，正式 `question_crops` 目录与 `question_crops.json` 才会进入原子切换阶段。已有正式结果先改名为同目录备份；发布任一步骤失败时恢复旧目录和旧清单，并清理临时文件及备份。

相同输入再次运行时，程序会比较完整清单，并逐张复核正式文件的尺寸、字节数和 SHA-256。完全一致则直接返回，不改动文件或清单时间戳。配置变化会重新生成整套结果；源页哈希变化不会静默复用。

## 输出清单

`question_crops.json` 记录版本、任务号、题目总数、所有来源页的路径、尺寸和 SHA-256，以及逐题信息：

- 题号、regions 和 composition
- 安全输出相对路径
- 输出宽高、字节数和 SHA-256
- `crop_status=generated`
- `review_status=pending_ai_review`
- 人工或自动发现的边界风险 warnings

## Web 安全接入

审核页按候选题号加载整题图。清单必须属于当前任务，题数和题号集合必须与候选题完全一致，每个文件必须使用固定命名、存在于 job 内、字节数和哈希一致且能作为声明尺寸的 PNG 打开。任一条异常时整份清单安全忽略，每题显示“单题原图尚未生成或清单异常”，其他审核内容继续正常显示。

私有图片路由只放行 `render_manifest.json`、`figure_assets.json` 和完整有效 `question_crops.json` 中明确列出的 PNG。未列出文件、非 PNG、绝对路径和路径穿越均不对外提供。

## 复核责任

程序验收只能证明文件、坐标、哈希和清单一致，不能替代视觉语义复核。生成后仍需由独立复核者确认每张图片包含题号、完整题干、全部选项或小问和作答所需图形，同时没有混入下一题、页眉页脚、广告或二维码。审核通过前保持 `pending_ai_review`。
