# PDF 导入与审核流程

## 一、当前阶段

本阶段只负责安全接收本地 PDF、校验输入、计算 SHA-256、去重、私有归档，并创建后续处理所需的导入任务。程序不会读取或输出 PDF 正文，不执行 OCR、公式识别或自动切题，不调用云端模型，也不访问网络。

调用方提供本地 PDF 路径、地区代码、可空年份、考试类型代码、试卷名称和可选页码范围。地区与考试类型必须来自数据库中已启用的字典。页码范围仅接受单页形式或闭区间形式，例如 3 或 1-6。

## 二、校验与归档

入口先确认路径存在且为普通文件，再检查大小、大小写不敏感的 `.pdf` 扩展名和 `%PDF-` 文件头。年份有值时必须是 1900 至 9999 的整数。试卷名称不得为空。

程序对输入文件计算 SHA-256。首次出现的哈希归档到私有数据根目录下的 `raw_papers/<地区>/<年份或unknown>/`。归档名由年份、地区、考试类型和哈希短前缀组成，不采用用户原文件名，因此原文件名中的路径片段不会影响保存位置。数据库只保存相对于私有数据根目录的 POSIX 相对路径。

复制先写入目标目录中的临时文件，写完后刷新到磁盘并重新计算哈希。只有复制前后哈希一致，才使用原子改名生成最终文件。相同 SHA-256 再次导入时复用已有 `source_papers` 记录和归档文件，但每次请求仍创建独立的 `import_jobs` 记录，以保留不同页码范围或不同处理批次。

## 三、数据记录

`source_papers` 保存不可变的原始试卷事实，包括完整 SHA-256、字节数、来源文件名、归档相对路径、地区、年份、考试类型、试卷名称和创建时间。SHA-256 与归档路径均唯一。

`import_jobs` 关联一个原始试卷，保存页码起止、状态、错误信息、创建时间和更新时间。首版状态受数据库约束，只允许以下值：

- `pending`：已可靠归档，等待后续处理。
- `processing`：后续 OCR 或拆题处理正在执行。
- `needs_review`：机器处理完成，等待人工核对。
- `completed`：人工审核和本批处理均已完成。
- `failed`：处理失败，错误信息应记录可操作的原因。

本阶段创建的任务固定从 `pending` 开始，不负责推进其他状态。

## 四、后续 OCR、拆题与人工审核

后续处理器必须以 `import_jobs.id` 为任务入口，通过 `source_paper_id` 找到原始归档文件，不得重新依赖用户最初提供的绝对路径。处理器应只读取原始 PDF，不修改归档原件。

计划中的后续顺序是：领取 `pending` 任务并置为 `processing`，按任务页码范围读取归档 PDF，执行页面提取、OCR、公式识别和候选切题，将候选题及定位信息写入待审核区，然后置为 `needs_review`。人工需要核对题目边界、正文、公式、配图、答案、来源页码和分类标签。审核通过后才能生成正式题目记录并把任务置为 `completed`；无法自动恢复的问题置为 `failed` 或退回 `needs_review`。

## 五、私有数据与 GitHub 边界

原始 PDF、运行数据库、临时复制文件和后续 OCR 中间产物都属于私有数据，不进入 GitHub。默认数据库位于 `data/private/question-bank.db`，默认原始归档位于 `data/private/raw_papers/`。项目的 `.gitignore` 已忽略 `data/private/*`、PDF 和常见 SQLite 文件，只保留私有目录占位文件。

GitHub 中只保存程序、数据库结构、无版权测试字节和技术文档。测试使用系统临时目录生成最小 PDF 字节，不读取真实试卷，也不应在 `data/private` 留下文件。

## 六、失败恢复与安全重试

复制中断、复制后哈希不一致、数据库约束失败或事务失败时，程序回滚数据库事务，并删除本次创建的临时文件和最终文件，不留下半条 `source_papers` 或 `import_jobs` 记录。临时文件与最终文件位于同一目录，以保证原子改名有效。

若目标归档已存在，程序必须确认其哈希与输入一致后才能复用。若数据库已有来源记录但归档文件缺失、越界或哈希不一致，程序拒绝创建新任务，等待人工从可信来源恢复原始文件。恢复完成后可用同一输入安全重试。

数据库操作开启外键约束。路径同时受业务层的根目录包含检查和数据库相对路径约束保护，禁止绝对路径、反斜杠及 `..` 路径片段，任何文件都不得写到 `raw_papers` 边界之外。

## 七、命令行入口

命令行模块为 `python3 -m src.importing.pdf_intake`。必填参数为 PDF 路径、`--region`、`--exam-type` 和 `--paper-name`；可选参数为 `--year`、`--pages`、`--database` 与 `--private-storage-root`。后两个参数用于隔离测试或明确选择私有数据位置。

成功时标准输出只有一行简洁 JSON，字段包括 `source_paper_id`、`import_job_id`、`stored_path`、`sha256`、`deduplicated` 和 `status`。输出不包含 PDF 正文或输入文件的绝对路径。
