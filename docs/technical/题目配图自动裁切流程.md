# 题目配图自动裁切流程

## 目标与边界

`src.processing.figure_crop.crop_figure` 从导入任务的原始页面 PNG 自动生成题目配图或审核证据。原始页面只读，裁切结果和 `figure_assets.json` 均位于 `data/private/processing/import_job_<id>/`，不进入正式题库，也不修改候选题或 AI 审核数据。

资源分为两类：

- `question_figure`：题目作答所需的正式配图，初始状态为 `pending_ai_review`。
- `review_evidence`：只帮助辨认原文的审核证据，状态为 `review_evidence`，不能作为题目配图。

## 调用方式

调用方传入 job 目录、源 PNG、输出相对路径、像素或归一化裁切框、题号、页码和用途。`margin` 可增加四周留白，触及页面边界时自动截断。像素框采用 Pillow 约定的 `(left, top, right, bottom)`，右、下边界不包含在结果内。

审核证据可通过 `processing` 选择：

```python
{"variant": "enhanced", "scale": 2, "contrast": 1.2, "sharpen": True}
```

只允许 2 倍或 3 倍增强（原色版为 1 倍）。处理仅包含 Lanczos 放大、对比度调整和非锐化遮罩，不做生成式补全、去水印或字符恢复。

## 安全与一致性

模块验证源文件存在、扩展名和实际格式均为 PNG，且源图位于指定 job 内；验证坐标、边界、最小尺寸、题号和页码。输出只允许 job 内的 `assets/` 或 `review/` PNG 相对路径，拒绝绝对路径、反斜杠和 `..`。

图片先写入同目录临时文件，重新打开验证格式与尺寸后用原子改名发布；失败会清理临时文件和未写入清单的成品。清单同样采用临时文件、落盘和原子改名。

每项资产记录源页及其 SHA-256、像素/归一化裁切框、输出尺寸、字节数、SHA-256、处理配置和审核状态。相同题号、用途、输出和处理配置重复运行时直接复用经校验的结果；若源页哈希、裁切框或成品哈希变化，则拒绝静默复用。

## Web 展示

私有图片路由只合并放行 `render_manifest.json` 中的原始页 PNG 与 `figure_assets.json` 中通过安全校验的 PNG。审核页按题号关联资产：正式配图直接展示“配图待AI复核”，审核证据提供原色与增强高清入口，并固定提示“增强仅用于辨认，不代表恢复原文”。

## 审核流程

1. 程序依据页面视觉布局定位并裁切。
2. 检查题干/图形完整性、相邻题污染、像素尺寸、文件大小和哈希。
3. 对边界不理想的资产调整坐标并重新生成。
4. AI 初审后仍需由 Hermes 独立视觉复核；本模块不会自动把状态改为正式通过。
